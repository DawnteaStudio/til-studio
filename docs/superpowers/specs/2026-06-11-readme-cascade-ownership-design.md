# README Cascade Ownership Design

## Goal

Keep every repository README consistent when a lowest-level note is added or deleted, without letting til-studio and the TIL GitHub Action rewrite the same README structure in incompatible formats.

## Ownership

til-studio owns repository publication and hierarchy indexes.

- A note save updates the source README, topic README, area README, and root README in the same atomic Git commit.
- A note deletion updates the same hierarchy in the same atomic Git commit.
- When the deleted note was the final content below a Studio-created source, topic, or area, Studio removes that empty managed README and removes its link from the parent README.
- Existing README prose is preserved. A README is deleted only when it contains no repository content and matches the minimal structure generated by Studio.

The TIL GitHub Action is a fallback reconciler for changes made outside Studio.

- It updates only the managed learning-log block between `til-studio:learning-log` markers.
- It never inserts directory guidance or writing principles into a non-empty README.
- It creates a minimal source README only when no source README exists and note or src content exists.
- It does not own topic, area, or root indexes.

## Add Flow

For a new path such as:

`area/topic/notes/source/note/entry.md`

Studio publishes one commit containing:

1. the note;
2. `area/topic/notes/source/README.md`;
3. `area/topic/README.md`;
4. `area/README.md`;
5. `README.md`.

The Action may run afterward, but its reconciliation must produce no additional commit when the source learning log is already current.

## Delete Flow

Studio calculates the repository path set after deletion before rendering READMEs.

- If note or src content remains in the source, Studio keeps the source README and refreshes its learning log.
- If no note or src content remains, Studio deletes a source README that is fully Studio-managed and removes the source from the topic index.
- If no note sources or theory documents remain under a Studio-created topic, Studio deletes that topic README and removes the topic from the area index.
- If no publishable topics remain under a Studio-created area, Studio deletes that area README and removes the area from the root index.
- If a topic or area README contains user-authored prose outside its managed block, Studio preserves the README and renders the managed block as empty.

The Action observes the final source state. If the source directory no longer exists, it makes no change.

## Managed README Detection

Deletion safety is based on content, not only the file path.

- A source README is removable only when it matches the Studio-generated source template plus its managed learning-log block.
- A topic README is removable only when it contains the generated heading and managed topic-index block without additional prose.
- An area README is removable only when it contains the generated parent link, heading, and managed child-index block without additional prose.
- The root README is never deleted.

Whitespace differences do not make an otherwise generated README user-authored.

## Error Handling

- Malformed managed markers fail the save or Action clearly instead of rewriting ambiguous content.
- A concurrent `main` update follows the existing non-force atomic publication and bounded Action retry behavior.
- A deletion request for an absent note keeps the existing conflict/not-found behavior.

## Verification

- A change-planner regression test proves a final note deletion removes a fully managed source, topic, and area README and updates the root README.
- A preservation test proves user-authored topic and area prose survives the final note deletion.
- An Action regression test proves a Studio-formatted source README receives only a learning-log replacement and no duplicated guidance.
- An Action test proves an already current Studio source README produces no file change.
- Full til-studio tests, lint, and build pass.
- Full TIL Node tests pass.
- A live smoke test creates a new lowest-level note and verifies source, topic, area, and root updates.
- The live smoke test deletes that note and verifies the temporary hierarchy and all parent links disappear.
